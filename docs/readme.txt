
1) 키오스크에 들어가는 장비 제어 UI
> 초기화면은 로고가 계속 움직임
> 화면의 특정 부분을 여러번 누르면 비밀번호 입력화면 뜸
> 비밀번호를 입력하면 서버에 확인 요청하고 응답 받아 처리 (리턴 > json)
> 접속이 완료되면 1,2,3,4 영역의 현재 볼륨을 확인할 수 있음 (리턴 json에 담아 보내주면 될듯)
> 각 영역를 클릭하면 기본, 행사PPT, 테마, 작품 을 선택할 수 있고
> 기본과 작품은 1개, 행사PPT는 행사/테마 각 5개, 테마는 5개 중에 하나를 선택할 수 있음

> 볼륨을 조절하면 /1/volume/0.7 이런식으로 전달 (1번 영역의 볼륨을 70%로)
> 볼륨은 전달할때 cms 서버에도 전송함
> cms에서 볼륨이 수정되면 소켓으로 전달받아서 osc로 다시 전송
> cms에서 전원제어를 선택하면 소켓으로 전달받아서 osc로 전송 (/1/power/off)
> 키오스크에서 전원제어를 선택하면 cms 서버로 전송해서 소켓으로 다시 전달 받아 osc로 전송

> osc의 명령은 바뀔 수 있기 때문에 command.json을 따로 만들어서 관리



2) 2대의 PC(아레나)에 들어가는 다운로더
> 다운로더를 실행하면 cms에서 파일 리스트 json을 다운
> 해당 json을 분석하여 지정된 폴더에 파일 저장
> 저장이 완료된 다음 curl 혹은 osc로 아레나에 파일 정보를 저장
> layers/3/clips/3/clear > 3번 레이어의 3번 클립 초기화
> layers/3/clips/3/open > 3번 레이어의 3번 클립에 지정

> 지정될 레이어와 클립 번호 정보가 있어야 하고, 그 정보를 cms에서 파일 리스트 json에 담아줘야 할 것 같음
> 다운로드할 때 ppt 파일의 경우에는 아레나에 등록이 안될거 같은데 어떻게 해야하는지?



3) file_list.json 설명
> header의 code가 SUCC일때만 진행
> list를 파싱해서 다운로드 처리
> type이 contents인 경우 file_url를 layer와 clip으로 구분하여 header의 base_path를 기준으로 저장
> type이 ppt인 경우 file_url를 layer와 clip으로 구분하여 base_ppt_path 기준으로 저장
