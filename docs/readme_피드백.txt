2) 2대의 PC(아레나)에 들어가는 다운로더
> 다운로더를 실행하면 cms에서 파일 리스트 json을 다운
   (소켓(UDP)로 업데이트 신호 전송시 다운로더 실행 다운로드 완료후 레이어 클립에 저장 후 CMS에 완료 호출)
> 해당 json을 분석하여 지정된 폴더에 파일 저장
   (ok)
> 저장이 완료된 다음 curl 혹은 osc로 아레나에 파일 정보를 저장
> layers/3/clips/3/clear > 3번 레이어의 3번 클립 초기화
> layers/3/clips/3/open > 3번 레이어의 3번 클립에 지정
   (총 4개의 레이어와 각 레이어별 10개의 클립으로 구성되어있고 총40개 리스트에서 순차적으로 레이어와 클립에 등록 예를 들어 총 파일이 10개면 1번파일 1번 레이어에 1번 클립, 2번은 파일은 2번 레이어에 1번 클립, 3번 파일은 3번 레이어에 1번 클립,4번 파일은 4번 레이어에 1번 클립,5번 파일은 1번 레이어에 2번 클립,6번 파일은 2번 레이어에 2번 클립 ....이렇게 순차 적용)

> 지정될 레이어와 클립 번호 정보가 있어야 하고, 그 정보를 cms에서 파일 리스트 json에 담아줘야 할 것 같음
   (위에 내용되로 이부분은 필요없어 지긴 하는데 만약 CMS 에서 json 으로 보낼때 위 부분을 처리해서 방식 으로 변경하면 필요함 이건 너랑 나 둘중에 누가 쉬운지 보고 결정)
> 다운로드할 때 ppt 파일의 경우에는 아레나에 등록이 안될거 같은데 어떻게 해야하는지?
   (작품 파일 외에 기본,ppt,테마는 파일 없어서 상관없음 )
